# report-pptx — 종목 리서치 .md → 디자인된 PPTX

리서치 마크다운을 읽고 **구조화된 JSON payload**로 추출한 뒤, 고정 디자인 렌더러
(`scripts/build_pptx.py`)로 PPTX를 생성한다. 디자인·폰트·슬라이드 순서는 스크립트가
강제하므로, 너의 역할은 **내용 추출·요약·검증**이다.

## 절차

1. **입력 확인**: `reports/{종목명}.md`를 찾는다. 없으면 `outputs/reports/`,
   `outputs/research/` 순으로 탐색한다 (같은 종목 파일이 여러 개면 종합).
   그래도 없으면 사용자에게 파일 위치를 묻는다.
2. **md를 읽고 payload JSON 작성** (스키마는 아래). 임시 파일은
   `outputs/research/_pptx_{종목코드}.json`으로 저장한다.
3. **차트 데이터(선택)**: `outputs/research/{종목코드}_technical_*.csv`가 있으면
   날짜·종가 컬럼을 읽어 `price.chart.dates/values`에 넣는다 (250포인트 초과 시
   균등 다운샘플은 스크립트가 처리). CSV가 없으면 차트 생략 — 텍스트만으로 구성된다.
4. **실행**:
   ```
   python ${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/scripts/build_pptx.py <payload.json> reports/{종목명}.pptx
   ```
5. **검증**: stdout의 "OK: ... (슬라이드 7장)" 확인. `[경고]`(매수/매도·목표가 표현
   감지)가 나오면 해당 문구를 payload에서 수정 후 재실행. 출력 파일 존재를 확인하고
   경로를 사용자에게 보고한다.

## 콘텐츠 규칙 (payload 작성 시 반드시 준수)

- **출처 없는 숫자는 싣지 않는다.** 모든 수치 문장 끝에 `(출처, 기준일)`을 붙인다.
  md 원문에 출처가 없는 수치는 슬라이드에서 제외한다.
- **매수/매도·목표가 단정 금지.** "목표가 X원", "사야 한다" 류는 쓰지 않는다.
  증권가 보도 인용이 필요하면 "~로 보도됨" 수준으로만. (스크립트가 금칙어를 재검사한다)
- **표 잘림 방지**: 재무 표는 **데이터 최대 7행 × 6열**로 직접 추려서 넣는다
  (우선순위: 매출액·영업이익·순이익·이익률 × 최근 3개년 + 최근 분기).
  스크립트도 7행 초과분을 자동 절단하지만, 의미 있는 행이 잘리지 않게 네가 먼저 고른다.
- **불릿 분량**: 슬라이드당 최대 5개, 1개당 한 문장(~90자). 길면 요약한다.
- 문체는 "~입니다" 체. 마크다운 기호(`**`, `#`)는 payload 텍스트에 넣지 않는다 (그대로 출력됨).

## payload JSON 스키마

```json
{
  "stock_name": "삼성전자",
  "ticker": "005930",
  "date": "2026-06-11",
  "subtitle": "한 줄 부제 (선택)",
  "overview":   { "bullets": ["..."], "source": "DART, 기준일 2026-06-11" },
  "financials": {
    "table": { "headers": ["항목", "FY2023", "FY2024", "FY2025", "1Q26"],
               "rows": [["매출액(조원)", "...", "...", "...", "..."]] },
    "bullets": ["표 해설 1~3개"],
    "source": "DART 사업보고서·분기보고서"
  },
  "price": {
    "chart": { "dates": ["2025-06-11", "..."], "values": [56900, ...],
               "title": "종가 추이 (원)" },
    "chart_image": "이미 만든 PNG 경로 (chart 대신 사용 가능, 선택)",
    "bullets": ["추세 요약 3~5개"],
    "source": "FinanceDataReader, 기준일 YYYY-MM-DD"
  },
  "news": { "bullets": ["핵심 이슈 3~5개, 각각 (매체, 날짜) 표기"],
            "sentiment": "중립(혼조, 강세 우위) — 근거 한 줄",
            "source": "웹 보도 종합" },
  "risks": [ { "title": "리스크 제목", "impact": "높음", "basis": "근거 한두 문장" } ],
  "risks_source": "리스크 매니저 보고서, 2026-06-11",
  "summary": { "line": "한 줄 종합 (단정 없는 판단 근거)", "source": "리드 종합" },
  "data_sources": ["DART OpenAPI 사업·분기보고서 (기준일 2026-03-31)",
                   "FinanceDataReader 일별 시세 (기준일 2026-06-11)"]
}
```

- `data_sources`(최대 6개)는 마지막 종합 슬라이드 하단에 "데이터 출처·기준일" 목록으로
  렌더링된다 — CLAUDE.md 가드레일(끝에 출처·기준일 목록) 충족용으로 **항상 채운다**.
- 표지의 "무료 공개 데이터 기반 학습용" 문구는 스크립트가 자동 삽입한다.

- `risks`는 최대 3개만 렌더링된다.
- `chart`와 `chart_image` 둘 다 없으면 가격 슬라이드는 불릿 전체 폭으로 구성된다.
- 표지·면주(footer)의 "학습용 분석, 투자 권유 아님" 문구는 스크립트가 자동 삽입한다.

## 디자인 (스크립트 고정 — 수정 불필요)

16:9, KB 옐로우 #FFBC00 포인트 + 다크그레이 #26282B/그레이 본문, 전 텍스트 맑은 고딕
(latin/ea/cs 모두 지정), 슬라이드 7장 고정 순서: 표지 → 종목 개요 → 재무 요약(표) →
가격/추세(차트) → 뉴스·심리 → 리스크 → 한 줄 종합.
